# Release Notes: 0.2.11

## Highlights

This release removes the starter dependency on local scaffold wrapper scripts. New starters call the package-owned scaffold bins directly through `npm run new-post` and `npm run new-page`.

## Major Themes

### Scaffold commands are package-driven

- Starter `package.json` now maps `new-post` directly to `anglefeint-new-post`.
- Starter `package.json` now maps `new-page` directly to `anglefeint-new-page`.
- The old starter-local `scripts/new-post.mjs` and `scripts/new-page.mjs` files are marked obsolete for starter sync.

This keeps the user-facing commands stable while letting npm package upgrades carry future CLI fixes.

## Migration Notes

Existing starter projects should migrate their package scripts with npm commands instead of editing JavaScript files manually:

```bash
npm install @anglefeint/astro-theme@latest
npm pkg set scripts.new-post="anglefeint-new-post"
npm pkg set scripts.new-page="anglefeint-new-page"
npm install
npm run new-post -- --help
npm run new-page -- --help
```

After migration, the daily commands remain:

```bash
npm run new-post -- my-post
npm run new-page -- projects --theme matrix
```

Users can also call the package bins directly:

```bash
npx anglefeint-new-post my-post
npx anglefeint-new-page projects --theme matrix
```

## Starter Sync

- Starter sync is required after publish so the `starter` branch dependency range, lockfile, npm scripts, and obsolete wrapper cleanup move together.
