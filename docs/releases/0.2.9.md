# Release Notes: 0.2.9

## Highlights

This release hardens the published theme's social metadata output and makes maintainer npm release dry-runs safer before publish.

## Major Themes

### Social metadata correctness

- Blog detail pages now pass existence-aware locale alternate URLs into the shared head renderer, so missing localized slugs fall back to the target locale blog index instead of advertising unavailable article URLs.
- Blog listing pages now apply the same alternate URL guard for paginated locale routes.
- Open Graph and Twitter image metadata now include image alt text, defaulting to the page title when a more specific value is not supplied.

### Release workflow guardrails

- `npm run release:npm` now checks the npm registry before packing or publishing and fails early when the workspace package version is not newer than the published latest version.
- The release script now uses a per-run temporary npm cache directory instead of a POSIX-only `/tmp` path.
- The release runbook now makes the intended order explicit: prepare and commit release state before running the publish dry-run.

## Migration Notes

- **From npm:** run `npm update @anglefeint/astro-theme` after `0.2.9` is published.
- **Content / config:** no `src/site.config.ts`, content schema, or route changes are required for this release.
- **SEO behavior:** existing pages receive the metadata hardening automatically through the shared theme head components.

## Starter Sync

- Starter sync is required after publish so the `starter` branch dependency range and lockfile move to `@anglefeint/astro-theme@0.2.9`.
- No starter-specific runtime or config migration is required.

## Validation Summary

- `npm run lint`
- `npm run check:docs`
- `npm run check`
- `npm run maintainer:sync-starter:check` (detected expected starter drift for README translations, preview images, package version, and lockfile)
- `npm run release:npm -- --dry-run`
