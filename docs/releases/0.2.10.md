# Release Notes: 0.2.10

> Update: `0.2.11` replaces the wrapper-copy migration path with package-owned scaffold scripts. Existing starter users should follow the `0.2.11` upgrade commands instead of copying local wrapper files.

## Highlights

This release fixes the published scaffold CLI entrypoints used by `anglefeint-new-post` and `anglefeint-new-page`.

## Major Themes

### Package CLI compatibility

- Added Node shebangs to the published scaffold CLI entry files so npm-generated bin shims execute them through Node on Windows, macOS, and Linux.
- Added direct `--help`, `-h`, and `help` handling to both package CLI commands.
- Kept starter wrapper commands aligned with the installed package path instead of a maintainer-only workspace path.

### Release guardrails

- `npm run check:scaffold` now verifies the scaffold CLI help path before testing file generation.
- `npm run check:pack-cli` now verifies that the packed npm tarball contains CLI entry files with Node shebangs.

## Migration Notes

- **From npm:** run `npm update @anglefeint/astro-theme` after `0.2.10` is published.
- **Starter wrapper users:** new starter checkouts receive the fixed `npm run new-post` and `npm run new-page` wrappers through the starter branch.
- **Existing starter users:** prefer the `0.2.11` migration, which updates package scripts to call `anglefeint-new-post` and `anglefeint-new-page` directly.
- **Content / config:** no `src/site.config.ts`, content schema, or route changes are required.

## Starter Sync

- Starter sync is required after publish so the `starter` branch dependency range and lockfile move to `@anglefeint/astro-theme@0.2.10`.
